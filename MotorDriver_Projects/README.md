# MotorDriver_Projects

## 概要
これまで作成した表面実装基板の設計データ、およびテスト用ファームウェアをまとめたディレクトリである。
モータドライバの開発コードネームが「Mars（火星）」であったことを発端に、以降製作された電磁弁駆動基板など周辺回路にも惑星の名前が付けられている。
各基板プロジェクトには、CADの元データのほか、閲覧用の回路図（PDF等）および基板レイアウト図が同梱されている。

## ディレクトリ構成とプロジェクトの変遷

### 1. Project_ver1
* **開発ツール:** EAGLE
* **`MD_Mars_ver1`**
    先輩が設計した初期型モータドライバ。メインマイコンに表面実装タイプのPICを採用しており、通信方式はSPIである。

### 2. Project_ver2
* **開発ツール:** EAGLE
* **`MD_Mars_ver2`**
    先輩の`ver1`をベースに、自身で再設計を行ったモデル。製作のしやすさや大会現場でのメンテナンス性を考慮し、マイコンを表面実装からDIP（ピン）タイプのPICへと変更した。通信方式はSPIを継承している。
* **`MD_Mars_study`**
    モータドライバの回路構成や仕組みを深く理解するために自作した学習用基板。部内のCNCルーターで切削加工して製作したが、設計したランド（銅箔部）が細すぎたため、大電流を流した際に焼損した。失敗からプリント基板におけるパターン幅の重要性を学んだ基板である。

### 3. Project_ver3
* **開発ツール:** EAGLE
* **アーキテクチャの刷新**
    「各モータドライバに1つずつマイコンを載せる」方式を廃止し、「1つのメインマイコンで複数のモータドライバを集中制御する」方式へと仕様を大きく変更した。PCのメモリのように、拡張基板に対してドライバを垂直に突き刺して使用する設計を採用している。以下の3つの基板から構成される。
    * **`MD_Mars_ver3`:** モータ駆動回路のみを切り出し、小型化を図ったドライバモジュール。
    * **`MarsBrain`:** メインマイコンからの制御信号を各モータドライバへ分配するための拡張基板。
    * **`BoneBoard`:** メインマイコンとモータドライバ群を相互に接続するためのバックプレーン（骨格）基板。

### 4. Project_ver4
* **開発ツール:** KiCad
* **アーキテクチャの再構築と最適化**
    設計ツールをKiCadへ移行。通信方式をノイズに強いCAN通信へと変更し、新たにゲートドライバICを採用したアップデート版である。
    * **`MD_Mars_ver4_prototype`**
        新規採用したゲートドライバの動作を検証するために、部内CNCで切削加工したテスト基板。この基板で確実な動作確認を行った後、本番機の実装へ進んだ。
    * **`MD_Mars_ver4.0.0`**
        メインマイコンにSTM32F303K8を採用した新基板。タクトスイッチやトグルスイッチを実装し、PCに繋がなくてもCANのID設定やモータの単体動作確認をハードウェア上で行えるよう設計した。
    * **`MD_Mars_ver4.1.0`**
        `ver4.0.0`の基板発注前にチーム内で協議し、回路配置や配線をさらに最適化した最終バージョン。実際の大会ロボットにはこの基板が搭載され、運用された。

### 5. Other (周辺回路基板)
モータドライバ以外の機能を持つ表面実装基板群である。
* **`Jupiter_12V`:** 12V駆動の電磁弁制御用基板。
* **`Jupiter_24V`:** 24V駆動の電磁弁制御用基板。
* **`PowerPlatform_24Vto5V`:** 24VのLiPoバッテリー電源から5Vへ降圧するための電源変換基板（実戦での使用実績はなし）。

## Firmware (テスト用プログラム)

当時のモータドライバ開発は複数人での分業プロジェクトであったため、本ディレクトリに含まれるファームウェアは自身が検証用に担当した部分のみとなっている。

* **`Mars_ver4.00_Test`**
    `MD_Mars_ver4_prototype`基板を用いて、新たに導入したゲートドライバおよびモータ駆動のハードウェア動作確認を行うために作成したテスト用プログラムである。
